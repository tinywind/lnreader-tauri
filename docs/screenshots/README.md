# Screenshot Capture Guide

Use this guide when preparing README screenshots. The goal is to show the app's
core value without exposing copyrighted novel text.

## Capture Set

Capture these three README images:

1. `library.png`
   - Size: `lg`.
   - Show the Library route with at least one public-domain demo book.
   - Include category controls, progress, unread/download badges, and the dense
     list or grid state.
2. `browse.png`
   - Size: `lg`.
   - Show Browse with a configured repository or installed source list.
   - If possible, include global search results for the demo book title.
3. `reader.png`
   - Size: `xs`.
   - Show paged reader mode with readable typography and visible reader chrome.
   - Avoid showing more text than needed to demonstrate layout.

Optional support captures, only if the README needs more proof of responsive
behavior:

- `novel-detail.png`
  - Size: `md`.
  - Show Novel detail with metadata, library action, progress, and chapter list.
- `settings-data.png`
  - Size: `sm`.
  - Show Settings > Data with backup export/import and maintenance actions.

## Legal Demo Books

Prefer Project Gutenberg because each book page clearly exposes free download
links and copyright status.

Recommended options:

- [Alice's Adventures in Wonderland by Lewis Carroll](https://www.gutenberg.org/ebooks/11)
  - Public domain in the USA.
  - Good for compact reader screenshots because chapters are short and familiar.
- [The Wonderful Wizard of Oz by L. Frank Baum](https://www.gutenberg.org/ebooks/55)
  - Public domain in the USA.
  - Good for library and reader screenshots because it has recognizable metadata
    and illustrations in the Project Gutenberg edition.

Do not use current commercial light novels, unofficial mirrors, fan
translations, or copyrighted web-novel pages in public screenshots.

## Size Mapping

- `xs`: phone portrait, around 390 x 844.
- `sm`: large phone or small tablet, around 640 x 900.
- `md`: tablet or narrow desktop window, around 900 x 900.
- `lg`: desktop, around 1440 x 900.

Prefer screenshots that show real app chrome and real UI states over cropped
mockups.
